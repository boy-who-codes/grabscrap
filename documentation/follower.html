KABAADWALA™

Hyperlocal Multi-Vendor Scrap Marketplace - System Flow & Architecture

User Registration & Authentication Flow

1
User Signup
User selects account type (Buyer/Vendor), provides email, password, and store name (for vendors)
↓
2
Email Verification
Celery sends verification email with activation link, fallback to Django console backend
↓
3
Login with 2FA
User logs in with email/password, receives OTP via email for 2FA verification
↓
4
Profile Setup Wizard
Multi-step form: Basic details → Phone verification → Store details → KYC documents (vendors only)
↓
5
Dashboard Access
User redirected to appropriate dashboard based on account type and profile completion status

User Types & Feature Flows

👤 Buyer

  • Browse scrap listings
  • Search by location & category
  • Place bids on items
  • Chat with vendors
  • Track order status
  • Rate & review vendors
  • Manage wallet & payments
  • View transaction history

🏪 Vendor

  • Complete KYC verification
  • Add scrap inventory
  • Set pricing & descriptions
  • Manage store profile
  • Respond to buyer inquiries
  • Process orders & payments
  • Track sales analytics
  • Withdraw earnings

⚙️ Admin

  • KYC approval/rejection
  • Monitor system performance
  • Manage user accounts
  • View transaction logs
  • Generate reports
  • System configuration
  • Dispute resolution
  • Content moderation

⚡
Celery + Redis

PostgreSQL
Production database with SQLite for development

Django Channels
WebSocket support for real-time features

Open-source mapping for location services
